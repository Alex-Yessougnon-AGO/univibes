# Procedure — Pre-Dispatch (social-copy)

> Load when SKILL.md routes to Pre-Dispatch. Encodes the read order, the Warm Start path (when a brief exists), the 5-question Cold Start prompt, the Write-back map, and the Cold-Start-under-`--fast` floor. Canonical Pre-Dispatch spec lives in [`../_shared/pre-dispatch-protocol.md`](../_shared/pre-dispatch-protocol.md) — this file is the social-copy-specific overlay.

---

## social-copy Pre-Dispatch shape

social-copy is a horizontal marketing skill: single-platform, single-market, single-brief (or topic) per invocation. Pre-Dispatch needs to resolve 5 dimensions before dispatching the copywriter-agent:

1. **Platform** — which of `tiktok / reels / shorts / x / linkedin`. Routing-only — drives which `platform-intelligence/[platform].md` file the agents read. Not persisted to experience/.
2. **Topic or brief** — either a path (`./docs/forsvn/artifacts/marketing/brief-shortform/[slug]/brief.md` or `./docs/forsvn/artifacts/marketing/campaign-plan.md`) OR an inline 1–2 sentence topic. Persisted to `experience/content.md` as a content event.
3. **Brand mode** — `founder` (single-author personal voice) OR `company` (multi-author brand voice). Persisted to `experience/brand.md` if novel for the project.
4. **Audience** — who this is for (role + one pain). Read from `research/icp-research.md` if present; otherwise asked + persisted to `experience/audience.md` if novel.
5. **Goal** — `awareness / engagement / click / save / share`. Determines CTA type generated by copywriter-agent. Default `engagement`. Persisted to `experience/goals.md`.

---

## Read order (in order)

1. **Auto-scan first** for context:
   - `research/product-context.md` (if present) — gives product, buyer, voice, primary CTA. Foundational input for copywriter-agent.
   - `research/icp-research.md` (if present) — gives personas + pain + voice + emotional drivers. Resolves the **audience** dimension without asking.
   - `brand/BRAND.md` + `brand/DESIGN.md` (if present) — gives brand_mode signal (founder vs company voice), archetype, lexicon, banned words. Resolves the **brand_mode** dimension without asking when the BRAND.md explicitly declares the mode.
2. **Pipeline (optional):** existing `docs/forsvn/artifacts/marketing/brief-shortform/[slug]/brief.md` if this is a follow-up to a short-form-brief run. If found AND the brief covers this platform + audience, this is a Warm Start with platform / topic / audience pre-populated — confirm + ask only about gaps (brand_mode + goal if not in brief).
3. **Pipeline (optional):** existing `docs/forsvn/artifacts/marketing/campaign-plan.md` if this is part of a campaign. Pull hook angle / audience / goal from campaign plan if present.
4. **Experience (read, don't ask):** `experience/content.md` (recent content events that might inform topic continuity), `experience/audience.md` (primary persona / pain points if persisted by a prior skill), `experience/brand.md` (brand_mode if persisted), `experience/goals.md` (current quarter goal if persisted).

After scan + read, present findings and ask only about the gaps.

## Warm Start (brief exists OR rich auto-scan)

Emit a confirmation prompt that summarizes what was found + asks only about gaps:

```
short-form-brief found at [path].
- platform → tiktok
- audience → indie founders, "outspent by bigger competitors"
- hook angle → fired $4K/mo agency for $200/mo stack
- goal → engagement (from brief)

Need before dispatching: brand_mode (founder / company) and variant_count
(default 2 — A/B; max 3)?
```

The Warm Start is intentionally short. When a `brief-shortform` artifact exists for the same topic + platform, skip all 5 Cold Start questions — the brief already locks platform / hook angle / audience / goal. Only brand_mode + variant_count need explicit confirmation.

---

## Cold Start (no brief, no scan signal — default for greenfield)

Emit one bundled prompt with 5 numbered questions:

```
social-copy generates platform-native social posts (hook + body + CTA) with
critic scoring against the 5-dimension rubric. To start:

1. **Platform** — tiktok / reels / shorts / x / linkedin?
2. **Topic or brief** — paste a brief path OR describe the topic in 1–2 sentences.
3. **Brand mode** — founder (personal voice) or company (brand voice)?
4. **Audience** — who is this for? (role + one pain, OR "use research/icp-research.md" if it exists)
5. **Goal** — awareness / engagement / click / save / share? (default: engagement)

Answer 1-5 in one response. I'll dispatch the 3-agent chain (copywriter →
format-checker → critic) and return the artifact with critic verdict.
```

If the user answers without enough signal to define topic or audience (e.g., "post about productivity" with no audience specificity), return `NEEDS_CONTEXT` per Completion Status — recommend either running `brief-shortform` first (if this is for video) or answering with a more specific topic + audience.

## Vietnamese-market auto-routing to vn-tone

When market signal is Vietnamese (any of: user explicitly says "Vietnamese," "VN," "Vietnam," "Việt Nam"; brief / topic supplies Vietnamese-language source text; `research/icp-research.md` declares persona geo as Vietnam; `experience/audience.md` declares Vietnam geo), Pre-Dispatch sets `--polish-chain vn-tone` as the default IF operator did not pass `--polish-chain` explicitly. Behavior:

1. **Operator passed `--polish-chain none` or `--polish-chain humanmaxxing`:** honor the operator's explicit choice. Do NOT override. (Operator may want raw English copy + manual VN translation downstream, or humanmaxxing-only for a non-VN region of a VN-bilingual market.)
2. **Operator passed `--polish-chain vn-tone`:** confirmed; proceed.
3. **Operator omitted `--polish-chain`:** override default `none` → `polish-vn`. Echo the override in the Warm/Cold Start confirmation: `Vietnamese-market detected; defaulting --polish-chain to vn-tone. Pass --polish-chain none to override.`

This implements the baseline SKILL.md auto-route ("Vietnamese-market copy auto-routes through vn-tone via --polish-chain vn-tone") and is the contract Anti-Pattern #13 (`anti-patterns.md` row "Vietnamese-Market Copy Without vn-tone Polish") guards. The detection happens in Pre-Dispatch (here) so the override is visible in the operator's Warm/Cold Start confirmation BEFORE dispatch begins.

## Cold Start STILL fires under `--fast`

Cold Start is a safety floor for social-copy. Under `--fast`:

- Orchestration collapses: skip format-checker revision loop (single-pass format check; if hard-cap violation, escalate FORMAT_FAIL to user without bouncing to copywriter).
- Critic gate collapses to single pass (no rewrite loop — there is no rewrite loop in social-copy baseline either; critic verdict ships with the artifact regardless).
- BUT the 5-question Cold Start STILL fires when context is missing. `--fast` does not authorize hallucinating audience or brand_mode — that would mismatch the brand voice (Anti-Pattern #5 — Brand-Voice Ignored) and waste the run. If platform + topic + audience can be inferred from auto-scan alone (e.g., short-form-brief + icp-research + brand/BRAND.md all present), the Warm Start path applies (no questions); otherwise Cold Start fires.

This matches the canonical mode-resolver contract: `--fast` skips orchestration weight, not the correctness floor.

## Write-back map

After Pre-Dispatch answers come in AND auto-scan completes, populate the per-domain experience files:

| Q | File | Key |
|---|---|---|
| 1. Platform | (routing only, not persisted) |
| 2. Topic or brief | `content.md` | `Content — recent topic` (append) |
| 3. Brand mode | `brand.md` if novel | `Brand — mode (founder/company)` |
| 4. Audience | `audience.md` if novel | `Audience — primary persona` (don't overwrite if icp-research already populated) |
| 5. Goal | `goals.md` | `Goals — recent campaign goal` |

**Q1 (Platform) is NOT persisted** — routing-only. Adding a `Platform — last used` key to experience/ would be net-new behavior and would mislead future skills (the user may run social-copy on TikTok one day and LinkedIn the next; "last used" is not a meaningful persistence target).

**Q4 (Audience) skip-if-exists rule** — if `research/icp-research.md` is present with a current persona that matches the topic, do NOT append to `experience/audience.md` (the canonical icp-research artifact is the source of truth; experience/ is a fallback for skills that haven't run icp-research). Only append when audience was supplied via Cold Start AND icp-research is absent.

## Anti-patterns in Pre-Dispatch

- **Skipping auto-scan to save time.** social-copy's auto-scan resolves audience + brand_mode in the typical case (when icp-research + brand-system have run). Without it, every invocation defaults to Cold Start, frustrating the operator. Always scan first.
- **Asking platform when the slash-command invocation already specified it.** The `argument-hint` declares `<platform>` as a positional argument; if the user typed `/write-social tiktok ...`, do NOT re-ask Q1. Cold Start is conditional on missing context, not unconditional.
- **Hallucinating brand_mode from product description.** If neither `brand/BRAND.md` nor user input declares `founder` vs `company`, ask Q3 explicitly. Defaulting silently to one or the other will trigger Anti-Pattern #5 (Brand-Voice Ignored) at critic time and waste the run.
- **Persisting Q1 (Platform) or Q5 (Goal as routing).** Platform is routing-only. Goal IS persisted (campaigns often have a consistent goal), but as the "recent campaign goal" key — not as a hard default that would override future runs.
- **Overwriting icp-research-populated audience.** If `research/icp-research.md` declares persona, do NOT append a contradicting persona to `experience/audience.md` based on a one-line Cold Start answer. icp-research is the canonical source; experience/ is fallback.
- **Running Cold Start under `--fast` without honoring it.** Cold Start fires under `--fast` regardless when context is missing — `--fast` only collapses orchestration after context is resolved (format-check revision loop collapse).

## Performance Grounding (pre-generation)

Before dispatching the copywriter-agent, ground on the operator's own channel history for the target platform:

```bash
bun scripts/query-performance.ts <platform> --json
```

Obey the emitted `empty | sparse | sufficient` state + `guidance` — empty store is the normal first-run state ("no own data yet — using priors"). Full read contract (precedence, the U12 direction-not-element-list rule, brand-floor supremacy): [`../_shared/performance-grounding.md`](../_shared/performance-grounding.md). Own data informs direction; it never becomes a prescriptive element list, and a brand floor always outranks it.

**Then emit the Recall line** (required output): state the own-data result with its real `n` + the `empty/sparse/sufficient` state, and read the loaded platform pack's `## Open Questions` to name **one** unknown as the experiment this run probes — see [`../_shared/performance-grounding.md`](../_shared/performance-grounding.md) § *The recall line*. The recall is shown in the post-copy output, not just obeyed; never fabricate `n` on an empty/sparse store, and drop the open-question part if the pack has none.
